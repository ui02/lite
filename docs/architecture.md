\# iolite Studio Architecture



\## 概要



iolite Studio は、Hugo + GitHub Pages をベースとした、自分専用の投稿プラットフォームです。



ブログ・SNS・画像管理を一元化し、「一度の投稿」で複数のサービスへ公開できることを目標とします。



\---

\# 管理画面



投稿画面では以下を入力する。



\- タイトル

\- 本文

\- タグ

\- カテゴリー

\- 画像（複数可）

\- 公開先選択



将来的には



\- 下書き

\- 予約投稿

\- AI補助



にも対応予定。



\---



\# Hugo



最終的な記事は Markdown として保存する。



記事は GitHub Repository が正本（Single Source of Truth）となる。



\---



\# 画像



画像は外部ストレージへ保存する。



Markdown には画像URLのみ保存する。



SNSへ投稿する際は、画像URLではなく画像ファイルを添付して投稿する。



\---



\# SNS



対応予定



\- X

\- Bluesky

\- Threads



投稿時に公開先を選択できる。



例



\- ☑ Hugo

\- ☑ X

\- ☑ Bluesky

\- ☐ Threads



\---



\# GitHub



GitHub Repository は以下を管理する。



\- Markdown

\- Hugo

\- テーマ

\- 設計書

\- GitHub Actions



\---



\# 設計方針



\## データは GitHub を正本とする



ブログ記事は GitHub に保存する。



CMS は GitHub を更新するためのツールである。



\---



\## 投稿は一度だけ



利用者は



\- 本文を書く

\- 写真を選ぶ

\- 投稿ボタンを押す



だけ。



以降は自動で処理する。



\---



\## スマホ対応



管理画面は PC・スマホの両方から利用可能とする。



旅行先からでも記事投稿できることを目標とする。
